Number input

The calculator-grade number field. It ships prefilled with an example value, clears fully on the first tap — and never again once you have typed something yourself — groups digits Indian-style as you type, and reads the number back in words underneath: 25,00,000 is 25 lakh. All three behaviours come from facet.js the moment the input carries data-number.

Live

Tap the field: the example clears in one go. Type a big number and watch the grouping and the words.

Snippet

<label class="field">
  <span class="field-label">Monthly investment</span>
  <span class="number-input">
    <input type="text" inputmode="numeric" value="25000" data-number
           data-tip="Amount in rupees. First tap clears the example.">
    <button type="button" class="btn btn-ghost btn-icon btn-small number-clear"
            aria-label="Clear" data-tip="Clear this field">&times;</button>
  </span>
  <span class="number-words" aria-live="polite"></span>
  <span class="field-hint">What you invest every month</span>
</label>

Reading the value in an app: facet.numberValue(input) returns the plain number behind the grouped text. The error state is the .field-invalid class plus a .field-error line replacing the hint.
