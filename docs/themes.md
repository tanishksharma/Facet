# Themes

## The four

- Default. Cool near-white and near-black; the accent is the ink itself; hairline borders, light shadows; dark is a near-black surface ladder.
- Velvet. Neumorphic matte material, one light source above, gold ink, serif display, tuned press physics.
- Aero. The Frutiger Aero era: aurora ground, glass gloss, translucent plastic, bubbly pills, wet sheen on headings; dark is ocean glass.
- Elegant. Roman. Cream and gold, serif display, carved elevation; dark is obsidian and gold.

## Mechanics

- One attribute on html switches the theme, layout included.
- Dark mode is its own attribute and composes with every theme.
- Both work on any element, flipping only that subtree.
- The URL carries theme, mode and language; shared links open identically.
- System dark preference is honoured when the page does not choose.
- The script tag boots the look before first paint; one call changes anything live, no reload.
- Switcher buttons are one attribute; an empty value means Default.
- A theme changes colors, faces, elevation style, scrollbars, form controls and the browser bar color.
- Markup never changes between themes.

## Tools

- Appearance panel. The settings sheet as a live customiser: theme, appearance, motion, sound, colors, fonts, density, width, text size, shape.
- Skin Lab. Every theme, light and dark, side by side, from identical markup.
- Style mixer. Restyle per token, carry it in the URL, copy the config.
- Custom accent. Override one rank in one block, page-wide or per subtree.
- Brand onboarding. One, two or three brand colors map onto the placement ranks; a single-color brand sets all three and lets placement carry the difference.
- Community themes. Build one, submit it, a marketplace lists approved ones.
- OS accent. The system accent color as an opt-in.
- Adaptive ink on glass is defined in Elements, decoration.
